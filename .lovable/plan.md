
## نظرة عامة
موقع متجر أعشاب باللغة العربية (RTL) بهوية إسلامية بصرية مستوحاة من الصور المرفقة (أخضر زيتي + بيج فاتح + خط مزخرف للعنوان). البيانات على Neon، الصور على Cloudinary، الذكاء الاصطناعي عبر Lovable AI، وكل المحتوى قابل للتعديل من لوحة تحكم محمية.

## البنية التقنية
- **الواجهة:** TanStack Start (React 19 + Vite) — يعمل على Vercel وLovable بدون مشاكل.
- **قاعدة البيانات:** Neon (PostgreSQL) عبر `@neondatabase/serverless` داخل server functions.
- **التخزين:** Cloudinary (رفع موقّع من الخادم، عرض من CDN).
- **AI:** Lovable AI Gateway (Gemini Flash) عبر server function streaming.
- **حماية لوحة التحكم:** كوكي جلسة موقّع (JWT بسيط) — بريد + كلمة مرور (مخزّنة كـ hash في الأسرار).
- **بصمة المستخدم:** fingerprint مولّد مرة واحدة ومحفوظ في `localStorage` + IndexedDB كنسخة احتياطية (يبقى ثابتاً عبر إعادة التحميل).

## مخطط قاعدة البيانات (Neon)
- `site_settings` (key, value JSONB) — اسم المتجر، رقم الواتساب، رقم الدعم، الموقع/البريد/الهاتف، برومت AI، الخ.
- `slides` (id, image_url, title, subtitle, cta_text, sort_order, created_at)
- `posts` (id, kind ['text'|'image'], title, body, image_url, pinned bool, created_at)
- `post_likes` (post_id, fingerprint) UNIQUE
- `post_comments` (id, post_id, fingerprint, author_name, body, reply_body NULL, reply_at NULL, created_at)
- `categories` (id, name, image_url, sort_order)
- `products` (id, category_id, name, price NULL, currency, quantity NULL, stock_status ['available'|'unavailable'|'limited'], image_urls TEXT[], description, notes, views_count, created_at)
- `product_views` (product_id, fingerprint, viewed_at) — لعدّ المشاهدات الفريد
- `ai_disclaimer_accepted` (fingerprint) — لإخفاء التحذير بعد الموافقة

## الشاشات
### 1. الشريط السفلي العائم (Glass / blur)
نفس التصميم في الصور: 4 أيقونات (الإعدادات / الذكاء الاصطناعي / المنتجات / الرئيسية)، خلفية شفافة مع `backdrop-blur`، حدّ خفيف، ظل ناعم. الأيقونة النشطة بلون أخضر والباقي رمادي.

### 2. الصفحة الرئيسية `/`
- هيدر: اسم المتجر «الطب الإسلامي البديل» بخط مزخرف (Aref Ruqaa / Amiri) + الوصف القصير. **بدون أي أيقونات.**
- سلايدر (Embla carousel) مع أسهم، تحميل كل صورة يعرض دائرة spinner.
- البوسترات: بطاقات بيضاء بزوايا مستديرة وظل ناعم، تظهر:
  - اسم الناشر «الطب الإسلامي البديل» فقط (بدون أيقونة، بدون ⋯).
  - تاريخ النشر بصيغة `يوم شهر سنة` بالعربية.
  - إن كان مثبّتاً: شارة «مثبّت» في الزاوية.
  - المحتوى (صورة أو نص بأسلوب اقتباس مع تأطير زخرفي خفيف).
  - أزرار: «أعجبني / تعليق / مشاركة» مع عدّاد اللايكات والتعليقات.
  - **تعليقات Facebook-style:** عند الضغط تنفتح Dialog/Sheet، يطلب الاسم أول مرة (محفوظ بالبصمة)، التعليقات تظهر مع ردّ الإدارة إن وُجد (تحت التعليق بشارة «المشرف»).
  - زر مشاركة: يستخدم Web Share API، يشارك النص + رابط الصورة (إن وُجدت).
- زر واتساب عائم (FAB) أسفل يمين.
- زر سلة الطلبات العائم (يظهر فقط عند وجود منتجات مضافة) فوق زر الواتساب.
- فوتر بمعلومات: الموقع، رقم التواصل، البريد (من `site_settings`).

### 3. صفحة المنتجات `/products`
- نفس الهيدر مع اسم المتجر.
- شريط بحث + زر تصفية (Sheet) — البحث في الاسم والوصف والصنف.
- خيارات الفلترة: الصنف، الحالة (متوفر/غير متوفر/كمية محدودة)، نطاق السعر، الترتيب (الأحدث/الأكثر مبيعاً/الأرخص/الأغلى).
- شريط الأصناف الأفقي (صور دائرية + الاسم).
- شبكة المنتجات (2-3 أعمدة): صورة، اسم، وصف قصير، عدّاد مشاهدات، سعر، **شارة الحالة** (متوفر أخضر / غير متوفر أحمر / محدود أصفر — كلها بـ blur وشفافية على الصورة).
- زر `+` يضيف للسلة (Local فقط، لا يمسّ DB).
- زر واتساب عائم + زر سلة (يعرض القائمة + زر «اطلب عبر واتساب» يبني رسالة جاهزة).
- النقر على بطاقة → `/products/$id`: معرض صور، تفاصيل كاملة، وصف، ملاحظات، زر «اطلب عبر واتساب». رفع عداد المشاهدات حسب البصمة.

### 4. صفحة الذكاء الاصطناعي `/ai`
- هيدر: «الذكاء الاصطناعي» نص فقط، بدون SVG، **بدون زر history، بدون زر +**.
- محادثة فقاعات (المستخدم يمين أخضر فاتح، المساعد يسار رمادي فاتح).
- Markdown مدعوم في الردود.
- عند أول رسالة فقط (إن لم تكن البصمة وافقت): Dialog تحذيري «قد يخطئ النموذج، استشر مختصاً»، زر «فهمت». بعدها يُحفظ في DB وlocalStorage فلا يظهر مجدداً.
- **لا حفظ للمحادثات** — تبدأ من الصفر في كل زيارة.
- يحقن system prompt من `site_settings.ai_prompt` + قائمة المنتجات الحالية كسياق.
- تدفّق (streaming) عبر server function.

### 5. صفحة الإعدادات `/settings`
- هيدر: «الإعدادات» نص فقط.
- 3 عناصر فقط:
  1. **حجم النص:** Slider (صغير/متوسط/كبير/أكبر) — يطبّق على body عبر CSS variable.
  2. **الدعم والمساعدة:** يفتح واتساب على رقم الدعم من DB.
  3. **عن التطبيق:** Sheet بنص قصير من DB.

### 6. لوحة التحكم `/admin`
- `/admin/login` — بريد + كلمة مرور، يقارن مع `ADMIN_EMAIL` و bcrypt hash، يصدر كوكي `admin_session` (JWT HS256 بـ secret خادم).
- `/admin` (محمية): سايد بار يميني بأقسام:
  - **الإعدادات العامة:** اسم المتجر، رقم واتساب الطلبات، رقم الدعم، الموقع/البريد/الهاتف.
  - **السلايدر:** إضافة/حذف/إعادة ترتيب صور (drag & drop).
  - **البوسترات:** إنشاء صورة/نص، تثبيت، حذف، **عرض التعليقات والردّ عليها**.
  - **الأصناف:** CRUD + ترتيب + صورة.
  - **المنتجات:** CRUD مع كل الحقول + **رفع جماعي للصور** (تحرّر بيانات كل منتج على حدة بعد الرفع).
  - **الذكاء الاصطناعي:** محرّر برومت + قواعد إضافية.
- كل الرفع يمر عبر server route → Cloudinary signed upload.
- تأثير «دائرة تحميل» (Spinner) أثناء كل رفع صورة في كل المواقع.

## نظام التصميم
- خطوط: `Aref Ruqaa` للعناوين الكبيرة (مزخرف)، `IBM Plex Sans Arabic` للنصوص.
- ألوان (oklch):
  - `--primary`: أخضر زيتي عميق (~ #4a5d3a)
  - `--primary-soft`: أخضر بيج فاتح للخلفيات
  - `--background`: عاجي دافئ (~ #fafaf5)
  - `--card`: أبيض ناصع
  - `--accent`: أخضر باهت
  - شارات الحالة: green/red/yellow بشفافية + blur.
- زوايا 2xl، ظلال ناعمة (`shadow-elegant`)، انتقالات framer-motion خفيفة على دخول البطاقات والـ Dialogs.
- RTL على مستوى `<html dir="rtl" lang="ar">`.

## المراحل (سأنفّذها بالترتيب — كل مرحلة قابلة للاختبار)
1. **الأساس:** dependencies، RTL، خطوط، نظام التصميم، schema DB، helpers (db.ts, cloudinary.ts, fingerprint.ts, auth.ts).
2. **التخطيط العام:** الشريط السفلي + الراوتر + الـ FABs (واتساب/سلة).
3. **الصفحة الرئيسية:** السلايدر + البوسترات + اللايك/التعليقات/المشاركة + الفوتر.
4. **صفحة المنتجات:** الشبكة + البحث + الفلترة + التفاصيل + السلة المحلية.
5. **الذكاء الاصطناعي:** المحادثة الـ streaming + التحذير لمرة واحدة.
6. **الإعدادات.**
7. **لوحة التحكم:** تسجيل دخول + كل أقسام الإدارة + رفع Cloudinary.
8. **بيانات معاينة** (تُحقن في DB كأمثلة قابلة للحذف لاحقاً).

## ملاحظات للسلامة والنشر
- جميع الأسرار (`DATABASE_URL`, `CLOUDINARY_*`, `ADMIN_*`) في الخادم فقط — لا تُسرَّب للمتصفح.
- المشروع يعمل على Vercel مباشرة (TanStack Start يدعم Vercel preset). سأضبط `vite.config` عند الحاجة.
- كل القراءات الثقيلة مع `Cache-Control` قصير، والبطاقات تُحمَّل بـ `lazy` + skeleton + spinner.

سأبدأ تنفيذ المرحلة 1 و 2 و 3 في الجولة القادمة بعد موافقتك.
